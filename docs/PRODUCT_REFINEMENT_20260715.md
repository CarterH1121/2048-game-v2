本轮以稳定 `master` 为基线，在 `codex/product-refinement` 上完成玩家体验精修。保持单文件 SPA、现有后端字段、数据库结构和认证边界不变，不部署生产。

- 账号道具先等待 `/api/items/use` 成功再改变棋盘和库存；失败时不制造本地成功。游客道具明确保持本地语义。

- 全球榜和公告继续使用真实只读接口；本地历史与全球榜分区展示。服务端昵称进入 HTML 前转义。

## 任务、成就与奖励结论

当前玩家端没有服务端任务进度、成就进度或领取奖励契约。因此本轮选择：

- 每日目标：本机每日进度，不发账户资产。
- 成就：本机可验证里程碑，不伪造领取和云同步。
- 签到：服务端记录与奖励返回，前端只展示真实结果。
- 道具：登录账户以服务端库存/消耗为准；游客为本机体验。

服务端可配置任务、成就和皮肤不能安全等同于玩家端已接入，需后续独立定义 ID、进度、领取幂等和迁移契约。

### 2026-07-16 V2.0 核心闭环补充

- 登录玩家任务已改为读取服务端奖励定义和领取状态；领取通过唯一奖励流水与库存事务完成，重复点击只返回已领取，不重复增加库存。
- 限时模式保存绝对截止时间，刷新、切后台和关闭页面重开都按实际流逝时间恢复，超时只结算一次。
- 离线游客可升级为 loopback 本地测试账号；同浏览器已完成对局和由其推导的任务进度进入事务迁移，昵称冲突或故障注入时不留下半成品账号。
- 本地测试账号恢复使用本机生成的账号密钥，服务端只保存哈希。它不是生产登录方案，不迁移任意客户端库存，也不提供跨设备棋盘、设置或成就同步。

## 发布边界与剩余风险

- `index.html` 与 `sw.js` 仍为同一发布/回滚单元，缓存版本随本轮 app-shell 变更提升。
- 真实生产写接口、真实账号、数据库落库、生产性能和生产发布未在本轮执行。
- 当前生产 HTTP 非安全来源不会启用 Service Worker；应用仍需诚实降级，离线 app-shell 只在未来安全来源下生效。
- 单文件 SPA 仍有维护成本；本轮未为重构而重构。旧的未初始化兼容代码可在后续行为快照充分时单独清理。

- 视觉证据：忽略目录 `output/playwright/refinement-before/`、`output/playwright/refinement-after/` 和 `output/playwright/acceptance/`。
