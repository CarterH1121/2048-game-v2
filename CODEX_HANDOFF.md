## 2026-07-15 本地收尾

- 移除了仓库部署脚本中的明文凭证和 `sshpass`，改为环境变量加 key-based SSH。
- localhost API 改为同源 `/api`，部署默认按当前 hostname 计算 API 地址；修复排行榜和日志模块的硬编码主机。
- API 非 2xx、超时和断网现在进入明确的本地游客模式，不再把临时本地 ID 伪装成已登录账号；游客签到/云同步给出清晰提示。
- 修复设置页调用不存在的 `API.login()` 所导致的运行时错误。
- 对公告与全球排行榜的服务端文本进行 HTML 转义，降低存储型 DOM XSS 风险。
- Service Worker 导航改为 network-first，缓存使用显式发布版本；API/跨域响应不进入 app-shell 缓存。
- 修复移动端安全区域、44px 顶部触控目标、随屏宽增长的棋盘、道具栏两端按钮裁切，以及短横屏棋盘/道具栏被截断的问题。
- 增加 favicon，消除首次加载的本地 404；允许用户缩放以改善可访问性。
- 建立可真正失败的 `npm test`，浏览器测试阻断全部非本机请求并使用本地 API mock。

## 已知风险与后续建议

1. 后端仍采用“昵称登录/注册”契约，不能等同于完整认证。需要与 `2048-admin` 的真实单体后端单独完成契约和认证治理。
2. 单文件 SPA 接近万行，维护风险高；后续可先做无行为变化的模块抽取方案和快照回归，不能直接重写。
3. 大量内联样式和脚本使严格 CSP 难以落地。本轮已封堵已确认的服务端文本注入点，但完整安全治理仍需系统性 DOM sink 审计。
4. Google Fonts 是跨域可选资源；离线时使用系统字体，视觉会有轻微差异。
5. 历史根脚本存在“发现失败仍输出审核通过”以及旧模块名断言。`npm test` 已不依赖这些结论；保留它们仅供历史对照。
6. 接管验收早期在拦截规则启用前，浏览器曾向当时配置的生产 API 发出一次昵称登录请求。未执行任何数据库回滚或其他生产写操作；如需确认影响，应由获授权人员检查后端用户/访问日志。
7. 首次断网创建的游客档案不会自动升级为云端账号；当前优先保证本地可玩且不误上报，后续需要产品定义“游客绑定/迁移”流程。
